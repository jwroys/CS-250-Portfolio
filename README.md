# CS 250 Portfolio Artifact

## Project Overview
This repository contains my Sprint Review and Retrospective from CS 250: Software Development Lifecycle. This project demonstrates my understanding of Agile practices including sprint planning, iterative development, and continuous improvement.

## Reflection

### How do I interpret user needs and implement them into a program? How does creating user stories help?
User needs are translated into technical work through user stories, which break down requirements into clear, testable goals. Writing user stories helps ensure development stays focused on delivering value rather than just implementing features.

### How do I approach developing programs? What Agile processes do I hope to incorporate into my future work?
I approach development iteratively by building small increments, testing frequently, and adapting based on feedback. Agile practices such as sprint planning, retrospectives, and backlog refinement help maintain flexibility while still delivering structured progress.

### What does it mean to be a good team member in software development?
Being a good team member means communicating clearly, accepting feedback, being accountable for deliverables, and supporting shared goals. Agile development depends on collaboration and transparency rather than isolated work.
